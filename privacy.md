# Privacy Policy for Google Calendar Reader

Effective date: March 2026

Google Calendar Reader is an NVDA add-on for Windows that allows users to sign in to Google Calendar and hear calendar events through speech output.

## What data the add-on accesses

Google Calendar Reader accesses Google Calendar data only after the user explicitly signs in with their Google account and grants permission.

The add-on accesses:

- calendar metadata, such as calendar names,
- calendar events, including event titles, dates, times, and all-day status.

## How the data is used

The data is used only to provide the core functionality of the add-on, including:

- reading upcoming events for today and future days,
- informing the user when an event is currently in progress,
- allowing the user to choose which calendars should be included in speech output.

## What the add-on does not do

Google Calendar Reader does not:

- create events,
- edit events,
- delete events,
- share calendar data with the developer,
- upload calendar data to any server controlled by the developer,
- use calendar data for advertising,
- sell user data.

## Local storage

The add-on may store the following files locally on the user's computer:

- `token.json` – a local Google OAuth token used to keep the user signed in,
- `settings.json` – local add-on settings, such as selected calendars and speech mode,
- `last_oauth_error.txt` – a local diagnostic file used for troubleshooting sign-in problems.

These files are stored only on the user's own device.

## Third-party services

Google Calendar Reader uses Google Calendar API to access calendar data in read-only mode.

Google's handling of personal data is governed by Google's own privacy policies and terms.

## User control

Users remain in control of their Google account access.

A user can stop the add-on from accessing Google Calendar at any time by:

- deleting the local token file,
- uninstalling the add-on,
- revoking the app's access in their Google account permissions.

## Data retention

The developer does not collect or retain users' calendar data on external servers.

Any locally stored authentication token or settings remain on the user's device unless the user removes them.
